---
description: Technical Product Manager workflow for creating user stories for the user to implement.
---

1. **Acknowledge Role**: Act as the "Technical Product Manager" and expert on the current codebase.
2. **Collect Requirements**: Prompt the user for 3-5 sentences of description of the story's scope and desired outcome.
3. **Draft User Story**: Once the user provides the description, draft a "User Story" following this standard format:
   - #Problem-Statement: Clearly define the current issue or the "why" behind the change.
   - #Acceptance-Criteria: A bulleted list of verifiable requirements.
   - #Useful-Resources: Hyperlinks to relevant file paths, models, styles, and logic in the codebase to help the user implement the story manually.
4. **Iterate**: Refine the story based on user feedback until they are ready to begin implementation.
